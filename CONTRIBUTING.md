# Contributing

Thanks for your interest in contributing to this project.

## Development Setup

1. Fork and clone the repository.
2. Create a feature branch from `main`.
3. Install Go 1.21+.
4. Run tests:

```bash
go test ./...
```

## Contribution Guidelines

- Keep changes focused and small.
- Follow existing code style and project structure.
- Add or update tests when behavior changes.
- Update docs when configuration or usage changes.
- Keep commits clear and descriptive.

## Pull Request Process

1. Ensure your branch is up to date with `main`.
2. Run tests and basic checks locally.
3. Open a pull request with a clear summary, motivation, and impact.
4. Link related issues when relevant.
5. Address review feedback promptly.

## Commit and Release Notes

- Prefer conventional commit style when possible.
- Include user-visible changes in release notes/changelog as needed.

## Questions

If you are unsure about a change, open an issue first to discuss approach and scope.
